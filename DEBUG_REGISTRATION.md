# Отладка регистрации

## Поле password_hash существует ✅

Поле уже есть в таблице, значит проблема в другом месте.

## Следующие шаги:

### 1. Проверьте логи backend после попытки регистрации

```powershell
docker-compose logs backend --tail 100
```

Ищите ошибки, которые появляются при попытке регистрации.

### 2. Проверьте через Swagger API

1. Откройте http://localhost:8000/docs
2. Найдите `POST /api/v1/auth/register`
3. Нажмите "Try it out"
4. Заполните данные:
```json
{
  "phone": "+79991234567",
  "email": "test@example.com",
  "first_name": "Тест",
  "last_name": "Тестов",
  "password": "Test123456",
  "telegram_id": null
}
```
5. Нажмите "Execute"
6. Посмотрите точный текст ошибки

### 3. Проверьте консоль браузера

1. Нажмите F12 в браузере
2. Откройте вкладку "Console"
3. Попробуйте зарегистрироваться
4. Посмотрите ошибки JavaScript

### 4. Проверьте Network вкладку

1. F12 → вкладка "Network"
2. Попробуйте зарегистрироваться
3. Найдите запрос к `/api/v1/auth/register`
4. Посмотрите ответ сервера

## Возможные причины:

1. Проблема с хешированием пароля (уже исправлено)
2. Проблема с валидацией данных
3. Проблема с подключением к БД
4. Проблема с CORS
