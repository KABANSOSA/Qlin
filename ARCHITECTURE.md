                        │ Webhooks / API Calls
                        │

        │      Backend API (FastAPI)     │
        │  ┌──────────────────────────┐ │
        │  │  Webhook Receiver        │ │

        │  └──────────────────────────┘ │

                │                   │

## Принципы интеграции

1. **Бот как внешний сервис** - бот не модифицируется, работает как есть
2. **Общая база данных** - веб и бот используют одну БД
3. **Webhook паттерн** - бот отправляет события в backend через webhooks
4. **API совместимость** - backend предоставляет те же endpoints, что использует бот
5. **State machine** - единая логика жизненного цикла заказов

## Поток данных

### Создание заказа (веб)
1. Клиент создает заказ через веб-интерфейс
2. Backend создает запись в БД (status: `pending`)
3. Backend отправляет уведомление боту через Telegram Bot API
4. Бот уведомляет уборщиков
5. Первый принявший получает заказ (race condition защита через Redis locks)

### Создание заказа (бот)
1. Клиент создает заказ через бота
2. Бот отправляет webhook в backend
3. Backend сохраняет заказ в БД
4. Дальнейший поток идентичен

### Принятие заказа уборщиком
1. Уборщик принимает заказ через бота
2. Бот отправляет webhook: `POST /api/v1/webhooks/telegram/order-accept`
3. Backend проверяет доступность заказа (Redis lock)
4. Backend обновляет статус: `pending` → `assigned`
5. Backend уведомляет клиента (веб + Telegram)

## Технологический стек

### Backend
- **FastAPI** - REST API framework
- **SQLAlchemy** - ORM
- **Alembic** - миграции
- **Pydantic** - валидация данных
- **Celery** - фоновые задачи
- **Redis** - кеш и очереди
- **PostgreSQL** - основная БД
- **JWT** - аутентификация

### Frontend
- **Next.js 14** - App Router
- **TypeScript** - типизация
- **Tailwind CSS** - стилизация
- **shadcn/ui** - UI компоненты
- **React Query** - управление состоянием
- **Zustand** - глобальное состояние
- **PWA** - прогрессивное веб-приложение

## Безопасность

- JWT токены с refresh механизмом
- Role-Based Access Control (RBAC)
- Webhook signature verification
- Rate limiting
- CORS настройки
- SQL injection защита (ORM)
- XSS защита (React)

## Масштабируемость

- Горизонтальное масштабирование backend
- Redis для распределенных locks
- Celery workers для фоновых задач
- Database connection pooling
- CDN для статических файлов
