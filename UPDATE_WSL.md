# 🔧 Обновление WSL для Docker Desktop

## Проблема
Docker Desktop требует обновления Windows Subsystem for Linux (WSL).

## Решение: Обновите WSL

### Способ 1: Через PowerShell (рекомендуется)

1. **Откройте PowerShell от имени администратора:**
   - Нажмите `Win + X`
   - Выберите "Windows PowerShell (Администратор)" или "Терминал (Администратор)"
   - Или найдите PowerShell в поиске, нажмите правой кнопкой → "Запуск от имени администратора"

2. **Выполните команду обновления:**
   ```powershell
   wsl --update
   ```

3. **Дождитесь завершения** (1-2 минуты)

4. **Установите WSL 2 как версию по умолчанию:**
   ```powershell
   wsl --set-default-version 2
   ```

5. **Перезагрузите компьютер:**
   ```powershell
   shutdown /r /t 0
   ```

### Способ 2: Через Docker Desktop

1. В окне Docker Desktop нажмите кнопку **"Restart"** (как показано на скриншоте)
2. Docker Desktop попытается обновить WSL автоматически

### Способ 3: Ручная установка WSL 2

Если автоматическое обновление не работает:

1. Откройте PowerShell от имени администратора

2. Выполните команды по порядку:
   ```powershell
   # Отключите WSL (если установлен)
   wsl --shutdown
   
   # Обновите WSL
   wsl --update
   
   # Установите WSL 2
   wsl --set-default-version 2
   
   # Установите дистрибутив Linux (если нужно)
   wsl --install -d Ubuntu
   ```

3. Перезагрузите компьютер

## Проверка после обновления

После перезагрузки:

1. **Запустите Docker Desktop**

2. **Проверьте версию WSL:**
   ```powershell
   wsl --version
   ```

   Должно показать версию 2.x.x или выше

3. **Проверьте статус Docker:**
   - Иконка в трее должна стать **зеленой**
   - В Docker Desktop не должно быть ошибок
   - Статус должен быть "Engine running" (не "Engine starting")

## Если проблема сохраняется

### Вариант 1: Переустановка WSL

```powershell
# Откройте PowerShell от имени администратора
wsl --unregister
wsl --update
wsl --install
```

### Вариант 2: Включите компоненты Windows

1. Нажмите `Win + R`
2. Введите: `optionalfeatures`
3. Найдите и включите:
   - ✅ "Подсистема Windows для Linux"
   - ✅ "Платформа виртуальной машины"
4. Перезагрузите компьютер

### Вариант 3: Обновите Windows

1. Откройте **Параметры Windows** (Win + I)
2. Перейдите в **Обновление и безопасность** → **Центр обновления Windows**
3. Проверьте обновления и установите их
4. Перезагрузите компьютер

## После успешного обновления

Когда Docker Desktop запустится без ошибок:

1. ✅ Иконка в трее зеленая
2. ✅ Статус "Engine running"
3. ✅ Нет сообщений об ошибках WSL

**Теперь можно запускать проект:**

```powershell
cd "C:\Users\бадан\OneDrive\Рабочий стол\QLINPRO"
docker-compose up -d
```

## Быстрая команда (скопируйте и выполните в PowerShell от имени администратора):

```powershell
wsl --update
wsl --set-default-version 2
shutdown /r /t 0
```

Эта команда:
1. Обновит WSL
2. Установит WSL 2 как версию по умолчанию
3. Перезагрузит компьютер
